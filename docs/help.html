Help

Short notes on using the builder.

Search template

A full web address with a single %s where the query goes, e.g. https://core.ac.uk/search?q=%s. Make these with the link converter.

Static links

A search can also be a plain link with no %s — for example a library homepage or a setup page. Paste a normal URL into the template field and it becomes a static link: it is always live and clickable on the page, with no query needed. Searches (with %s) still wait for a typed query; static links appear straight away. In the builder, each row shows a small search or link badge so you can tell them apart.

Links mode vs open-all mode

Links mode (default) is reliable everywhere: typing a query rewrites every button so each opens that engine's results in a new tab when clicked. Open-all mode adds buttons that try to open a whole section's engines at once; browsers may ask permission to allow pop-ups the first time.

Saving and reloading

Save Super Search page downloads the standalone HTML. Save config downloads a .json you can reload with Load config to keep editing. You can also Load page to pull an exported Super Search back into the builder. A working draft is also kept in this browser automatically.

Publishing

Rename your saved page to index.html and put it at the top of your GitHub Pages publishing source. Then bookmark it or set it as your browser homepage.

If a search will not convert

Some sites do not put the query in the address. Those cannot become a simple template — see Add a search for options (a site search usually works).
